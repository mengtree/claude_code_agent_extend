# Mijia Smart Home Skill

## 适用场景

当用户提到以下需求时使用本 Skill：
- 控制米家/小米智能家居设备
- 获取设备列表
- 读取设备属性（如亮度、开关状态）
- 设置设备属性（如开关、亮度、色温）
- 执行设备动作（如 `toggle`）

## 前置条件

1. 已安装依赖：`mijiaAPI>=3.0.5`
2. 首次使用需要扫码登录（`api.login()` 会触发）

## 执行流程

1. 若依赖未安装，先安装：
   - `python3 -m pip install --user mijiaAPI`
2. 列出设备确认目标（强制第一步）：
   - 使用 mijiaAPI 库列出所有设备
3. 获取目标属性当前值（强制第二步）：
   - 使用 mijiaAPI 库读取设备属性
4. 结合用户意图计算目标值并更新属性（强制第三步）：
   - 使用 mijiaAPI 库设置设备属性
5. 若用户明确要求执行动作，再执行动作：
   - 使用 mijiaAPI 库执行设备动作

## 设备操作决策规则

- 必须遵循顺序：`list_devices -> get -> set`
- `set` 前必须先执行对应 `get`，确认当前状态与属性名有效
- 更新值时要结合用户意图：
  - 相对指令（如"亮一点"）先读取当前值再计算新值
  - 绝对指令（如"亮度设为50"）也先读取当前值用于校验与回显
- 若设备名不唯一或匹配失败，先回到 `list_devices` 重新确认目标

## 参数规则

- 设备定位：优先使用 `--did`，其次 `--dev_name`
- `--value` 支持自动解析：
  - 布尔：`true/false`
  - 数字：`int/float`
  - 其他保持字符串

## 响应规范

执行后向用户返回：
- 实际执行的命令
- 核心结果（成功/失败）
- 如果失败，返回错误信息和可执行的下一步

如需要登录，应该同时返回登录提示里的二维码图片链接

## 常见命令

- 使用 mijiaAPI 库进行设备控制
- 首次使用需要扫码登录米家账号
- 登录后会自动保存 token，后续无需重复登录

## 安全与边界

- 仅在用户明确要求时执行写操作（`set`/`action`）
- 对于不明确的设备目标，先执行 `list_devices` 再确认
- 未完成 `get` 属性确认前，不执行 `set`
- 不输出敏感认证文件内容
